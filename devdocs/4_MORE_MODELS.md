## 追加计划：小样本多模型对比与强化

### 一、数据与特征工程强化
**目标**：从1500条日频原始数据中压榨出最大信息量，构建稳健的训练/验证/测试集。

1. **价格特征细化**
   - 生成滞后特征：`lag_1, lag_2, lag_3, lag_7, lag_14, lag_30`。
   - 滚动统计特征：过去7天、14天均价、标准差、最大值、最小值，用于捕捉短期趋势和波动。
   - 差分特征：一阶差分（`price_diff_1`）、周度差分（`price - lag_7`），突出短时变化。
   - 同比特征（可选）：`price - lag_365`，如果数据覆盖多个完整年份。

2. **气象特征强化**
   - 原始特征：日均气温（多城平均）、日降水量（多城平均）。
   - 滞后特征：滞后1、3、7天的气温和降水，反映天气对价格的延迟影响。
   - 累积/窗口统计：过去3天、7天累计降水量，过去7天平均气温、气温日较差（如果未来能拿到最高/最低温）。
   - 极端天气标记：当日降水 > 历史95%分位数记为1；连续3天高温 > 30°C（山东夏季常见）记为1。

3. **时间特征编码**
   - 星期几（one-hot 或 1-7 整数）、月份（1-12）、是否周末（0/1）。
   - 农历节气/节假日：春节、清明节、中秋节等单独标记，因对蔬菜需求冲击大。
   - 年周期编码：用 `sin(day_of_year * 2π/365)`、`cos(...)` 作为连续特征，帮助模型捕捉季节性。

4. **数据划分策略**（防止未来信息泄露）
   - 严格按时间顺序划分：
     - 训练集：2020-01-01 至 2024-12-31（约5年）
     - 验证集：2025-01-01 至 2025-09-30（约9个月，含春夏秋）
     - 测试集：2025-10-01 至 2026-05-21（约7.7个月，含冬季）
   - 对训练集计算标准化参数（均值、标准差），映射到验证和测试集。

5. **轻量数据增强**（仅用于训练集，避免增加虚假信息）
   - **SpanDrop**：随机选取连续2-5天，将其价格设为缺失，再用线性插值填充，生成新样本。保留率70%即可，不引入噪声。
   - **窗口长度扰动**：除固定的30天窗口外，额外使用28天和32天窗口构造少量样本，增强模型对输入长度的鲁棒性。
   - **不引入**：SMOTE、GAN、合成样本生成，避免污染时序结构。

### 二、核心模型实现与防过拟合体系
**原则**：每个模型保持简单，用不同角度捕捉规律，最后可做简单集成。

#### 1. LightGBM 主力模型
- **输入形式**：二维表格，每行一个样本，包含所有滞后/统计/时间特征。
- **参数配置**：
  - `objective='regression'` 或 `'quantile'`（若做分位数）
  - `max_depth=4`，`num_leaves=15`，`min_data_in_leaf=20` 限制复杂度
  - `lambda_l1=0.1`，`lambda_l2=0.5` 正则化
  - `learning_rate=0.03`，`n_estimators=300`，早停轮数`early_stopping_rounds=50`
- **输出**：
  - 点预测模型。
  - 分位数模型（单独训练 α=0.1 和 α=0.9），用于给出预测区间。
- **可解释性**：使用 `feature_importance` 排序，并用 `shap` 库画 summary plot（选部分样本即可，速度快）。

#### 2. 浅层 GRU 模型（文件1 推荐）
- **架构**：
  - 输入：30天 × 特征数（价格、气温、降水，可再加1-2个关键滞后）
  - 2层GRU，`hidden_size=64`，`bidirectional=False`
  - 层间 `Dropout=0.3`，输入 `Dropout=0.2`
  - 注意力层：取GRU所有时间步输出，用加性注意力（Bahdanau）学习权重，加权求和得到上下文向量
  - 输出：全连接层 `context → 1` （点预测）
- **防过拟合配置**：
  - 损失函数：Huber Loss（delta=1.0，平衡MSE和MAE）
  - 优化器：AdamW，`lr=1e-3`，`weight_decay=1e-4`
  - 早停：监控验证集MAE，`patience=15`
  - 梯度裁剪：`max_norm=1.0`
- **分位数输出**（借鉴文件2）：  
  复制点预测GRU，将最后一层改为同时输出三个值（P10, P50, P90），损失改为 Pinball Loss 之和（各分位数权重均为1/3）。这样只需训练一个模型即可获得不确定性区间。
- **注意力可视化**：推理时提取注意力权重，画出“哪个历史天对当前预测贡献最大”的热力图，用于案例分析。

#### 3. Prophet 季节分解模型
- 仅使用价格序列，将气温和降水作为外部回归量加入：
  ```python
  model = Prophet(daily_seasonality=True, weekly_seasonality=True, yearly_seasonality=True)
  model.add_regressor('temp_avg')
  model.add_regressor('precip_avg')
  ```
- 训练后输出趋势、周季节、年季节分解图，直观展示价格规律。
- **注意**：Prophet 对突变的捕捉较差，主要用于解释周期模式，不作为最终主预测源。

#### 4. 简单集成（可选）
- 将三个模型的点预测直接等权平均，或用 LightGBM 的点预测作为最终结果（因其在小表格数据上最稳），GRU和Prophet作为辅助解释和鲁棒性备份。

### 三、舆情偏置接口的具体实现
**目标**：在不实际训练NLP模型的前提下，让架构完整，展示前瞻性。

1. **特征占位列**：在训练数据中添加 `sentiment_score` 列，全部填 0。
2. **GRU/Prophet 输入维度扩展**：模型定义时包含该特征，但不影响现有训练（因为全是0）。
3. **Streamlit 界面添加“政策/舆情信号模拟”模块**：
   - 侧边栏滑块：`sentiment_score` 从 -1（极度利空）到 1（极度利好），用户可调节未来N天的信号值。
   - 模型预测时，将用户设定的分数替换占位0，生成新的预测曲线，与原始预测对比。
   - 路演说辞：“我们预留了实时政策/舆情接口，未来只需部署轻量NLP微服务，即可将文本情绪量化为日度分数，驱动动态价格调整。”
4. **历史事件标注**：在价格走势图上手工标注 3-5 个已知重大政策事件（如2022年某月抛储），说明模型在此期间的预测误差可能由未建模的政策冲击导致，体现思考深度。

### 四、评估与对比展示体系
**目标**：用严谨的指标和直观图表让评委信服。

1. **测试集指标计算**：
   - 对于所有模型，计算 MAE、RMSE、MAPE、SMAPE。
   - 对于含分位数的 GRU，额外计算：
     - PICP（预测区间覆盖率，理想值接近名义置信度80%）
     - CWC（基于区间宽度惩罚的覆盖率）
2. **可视化页面（Streamlit）**：
   - **Tab 1：预测总览**  
     默认显示 LightGBM 的未来7天预测曲线，叠加真实值（测试集）、预测区间。
   - **Tab 2：模型对比**  
     柱状图对比三个模型的三个指标，一目了然。
   - **Tab 3：季节性与趋势**  
     Prophet 组件分解图，并标注春节等特殊节日效应。
   - **Tab 4：天气冲击分析**  
     GRU 注意力权重热力图 + 某次暴雨前后预测与真实价格对比，突出模型对天气的敏感度。
   - **Tab 5：政策/舆情模拟**  
     上述滑块模块，展示动态调整效果。
3. **单案例深度分析**：
   - 选取测试集中一段典型波动（比如2025年10月一次降温后的价格上涨），用三个模型分别预测，对比谁的响应更快、更准，并用注意力图解释GRU的决策依据。
