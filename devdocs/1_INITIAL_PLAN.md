## 项目名称：开源齐民 —— 多源融合的农产品价格智能预测系统

### 1. 项目目标
- **预测对象**：山东省（示例）的 **黄瓜** 日度批发价（也可灵活换成其他省份/品种）。
- **数据源**：
  - 商务部各采样市场 **日度价格**（多个市场、多个品种，部分缺失）。
  - 国家重点农产品信息平台 **全国批发价**（统一口径）。
  - 气象数据（温度、降水）。
  - **预留**：政策/舆情偏置分数（当前用占位特征，未来可替换为 NLP 实时输出）。
- **模型**：基于 PyTorch 的 **LSTM + 注意力机制**（Bahdanau 风格或 simplified self-attention）。
- **部署**：单台 RTX 3050 笔记本，Streamlit 轻量可视化，支持预测未来 1-7 天价格。

---

### 2. 为什么选 LSTM + 注意力
- LSTM 能捕捉农产品价格的周期性、趋势性和天气滞后效应。
- 注意力机制可以自动学习过去哪些时间点（如一周前的暴雨、某天的政策冲击）对当前价格影响最大，**可解释性强**，适合路演展示权重分布图。
- 计算量完全在 3050 掌控范围内（4GB 显存，参数规模 < 10 万）。

---

### 3. 整体技术架构（数据流）
```
[商务部网页] → 爬虫 → 市场价表（多市场、多品种）
[全国批发价平台] → 爬虫 → 全国均价表
[Open-Meteo API] → 请求 → 气象表（温度、降水）
                          ↓
              数据清洗、对齐、合并为“某省某品种”宽表
                          ↓
         特征工程：滞后价格、滚动天气统计、时间特征
                          ↓
     [训练集/验证集/测试集] 按时间切分（80/10/10）
                          ↓
        LSTM + Attention 模型训练（GPU）
                          ↓
        评估、保存模型 → Streamlit 预测展示
```
**预留舆情接口**：在特征构建阶段，保留一列 `sentiment_score`，默认填 0，未来可接入 NLP 模块动态更新。

### 4. 数据工程详细设计

#### 4.1 数据爬取 (预计耗时 1.5-2h，含容错)
- **商务部市场价**：
  - 请求接口或解析 HTML，按市场、品种、日期抓取，保存为 `market_prices.csv`。
  - 列：`date, market, product, price`。
  - 爬取策略：设置超时 5s，失败重试 2 次，每次间隔 0.5s，确保 3 年数据完整。
- **全国批发价**：
  - 从 `pfsc.agri.cn` 或相应 API 获取日度全国平均价，保存为 `national_price.csv`。
  - 列：`date, product, avg_price`。
- **气象数据**：
  - 使用 Open-Meteo Archive API，根据目标省份省会坐标（如济南 36.67N, 116.98E），获取 `temperature_2m_max, temperature_2m_min, precipitation_sum`。
  - 直接生成 `weather.csv`，日频。

#### 4.2 数据合并与对齐 (预计 1h)
1. **选定目标**：假设为 **山东省-黄瓜**。筛选山东省内市场的黄瓜价格（如济南、青岛等批发市场）。
2. **多市场融合**：对省内所有市场的黄瓜价格，按日取 **中位数** 或 **均值** 作为“省内综合价”（可填充缺失，若当日无任何市场报价则用前向填充）。同时保留全国黄瓜批发价作为宏观特征。
3. **与天气合并**：将省内综合价、全国批发价、气温（取最高最低的均值）、降水合并到同一张表。
4. **缺失处理**：少量缺失价格用线性插值或前向填充；天气无缺失。
5. **异常值处理**：将价格变化超过 3 倍标准差的值截断或标记。

最终数据框：`date | local_price | national_price | temp_avg | precip | (sentiment_score_placeholder)`

#### 4.3 特征工程 (预计 1h)
构建监督学习样本：
- **输入序列长度**：T = 30 天。
- **预测目标**：未来第 1 天（或第 3、7 天）的价格。这里我们预测 **下一天价格**，可扩展到多步。
- **特征列**：
  - 原始值：`local_price`, `national_price`, `temp_avg`, `precip` （4 维）。
  - **可额外构建滞后特征**：过去 7、14 天均价差等，但为了避免输入维度过大，可只使用原始时序，让 LSTM 自行提取。为简单，我们使用多变量原始值作为输入，特征维度 d=4（或 5，加上预留的情绪分）。
- **数据标准化**：对每个特征进行 Z-score 标准化。

样本构建方式（滑动窗口）：
```
输入 X: [t-T+1, ..., t] 的 4 维向量
输出 y: 第 t+1 天的 local_price
```

#### 4.4 数据集划分
- 按时间顺序分：训练集（前 80% 时间）、验证集（中间 10%）、测试集（最近 10%），防止未来泄露。

### 5. 模型设计：LSTM + Attention

#### 5.1 网络结构
```
输入: (batch, T, d)   d=4 (价格、全国价、温度、降水)
↓
LSTM 层 (hidden=64, num_layers=2, batch_first=True, dropout=0.2)
   输出: (batch, T, 64) 的隐状态序列 h_t
↓
注意力层 (Bahdanau-style additive attention):
   - 计算每个时间步的注意力权重 α_t = softmax(v^T * tanh(W1*h_t + W2*h_last))
   - 其中 h_last 是 LSTM 最后时刻的隐状态（或取平均，或一个可学习的查询向量）
   - 加权求和得到 context vector c = Σ α_t * h_t
↓
拼接 context 与最后时刻隐状态 (或直接使用 context) → 全连接层 → 输出标量 (未来一天价格)
```
- **参数总量**：LSTM 64 隐层，两层，约为 66K；注意力及输出层很小。总参数量 < 100K，显存占用极低。
- **可解释性**：直接可视化注意力权重 α_t，显示哪些历史天对预测影响最大（如暴雨第 7 天权重高）。

#### 5.2 训练配置
- 损失函数：MSE
- 优化器：Adam，lr=0.001
- Batch size：32
- Epoch：100（早停 patience=10）
- 使用 3050 GPU 训练，预计耗时 20-30 分钟。

#### 5.3 评估指标
- RMSE, MAE（反标准化回原价格单位）。
- 对比基线：持久性模型（用昨天价格预测今天）、简单线性回归。

### 6. 预留舆论偏置的设计（创新展示核心）
在数据表中保留一个特征 `policy_impact`，目前全部填 0。但在代码和界面上明确体现：
- **数据处理层**：`add_policy_feature(date_range)` 函数返回全 0 列，未来可替换为调用 NLP 分析引擎。
- **模型输入维度**：`d=5`（加上该特征），当前模型训练时该特征不起作用，但不影响预测。
- **展示层**：在 Streamlit 界面单独设计一个 **“政策/舆情信号模拟”** 区块，允许用户手动输入一个未来日期的舆情分数（-1 到 1），观察对预测曲线的冲击。
- **路演说辞**：展示架构图，说明 “我们目前使用占位特征，但预留了标准接口，只需对接一个轻量级文本情感分析微服务，即可实时捕获政策新闻对价格的驱动，这是从‘历史拟合’到‘实时感知’的关键升级。”

### 7. 风险控制
- **爬虫失败**：如果某网站反爬严重，优先保证气象 API 和全国批发价，市场价用全国价替代，省内差异可暂忽略。
- **数据极稀疏**：若山东省市场日数据缺失超过 50%，退而求其次，**预测全国黄瓜批发价**（目标可微调），仍能体现多源融合。
- **GPU 不可用**：回退到 CPU 训练，数据量小仍可在 1 小时内完成。
- **注意力权重不明显**：在展示时选取一段典型暴雨后价格飙升的片段，手动解释其峰值权重，不依赖自动完美。
